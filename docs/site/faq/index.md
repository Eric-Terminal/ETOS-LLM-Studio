# 常见问题

## 这到底是手机 App 还是手表 App？

两者都是。  
ETOS LLM Studio 同时覆盖 iPhone 与 Apple Watch，而且很多能力会通过同步系统互通。

## 我是不是必须用 Apple Watch 才能用它？

不是。  
手表端是一个很重要的特色，但 iPhone 端本身已经是完整客户端。

## 第一次上手最容易卡在哪？

通常是提供商配置。  
建议先只接一个稳定模型，把文本聊天跑通，再继续开图像、语音、工具和代理。

## 为什么我感觉很多功能藏得很深？

因为项目功能确实很多，而且很多入口被放在设置页、工具中心或专门管理页里，而不是全部堆到聊天主界面。  
先看 [界面导览](/guide/interface-tour) 和 [隐藏技巧](/tips/hidden-gems) 会省很多时间。

## 这个项目只支持 OpenAI 吗？

不是。  
它支持多种接口格式，包括 OpenAI、Anthropic Claude、Google Gemini 以及兼容 OpenAI 的服务。

## 记忆是不是云端托管的？

不是。  
Embedding 可以调用云端 API，但向量数据库本身是本地 SQLite。

## 世界书和记忆应该怎么选？

- 需要长期检索的知识片段，优先放记忆
- 需要条件触发或场景注入的设定，优先放世界书

## Daily Pulse 是什么？

它是一套主动情报卡片机制，会根据你的配置和反馈整理出“今天可能值得看什么”，并支持提醒、保存、继续聊和转任务。

## 工具和 MCP 是必须用的吗？

不是。  
你完全可以把它当成一个纯聊天客户端来用；但如果你需要更强的工作流能力，工具系统会非常值得投入。

## 我一定要在手表上填 API Key 吗？

不需要。  
更推荐在 iPhone 端完成复杂配置，再通过同步带到 Watch。

## 遇到问题怎么排查更有效？

先分清是模型链路、工具链路、同步链路还是显示问题。  
如果已经怀疑是数据层面的问题，优先利用请求日志、局域网调试和反馈助手，而不是只盯着当前界面猜。
