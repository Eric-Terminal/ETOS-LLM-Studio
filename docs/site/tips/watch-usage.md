# Apple Watch 使用建议

ETOS LLM Studio 的手表端不是附属遥控器，但它也不应该被当作“所有事情都在这里完成”的唯一入口。  
最舒服的体验通常来自明确分工。

## 更适合在 Watch 上做的事

- 快速发起对话
- 接收并查看 Daily Pulse
- 继续之前已经存在的会话
- 用同步过来的配置做轻量交互
- 在碎片时间里完成短轮聊天

## 更适合在 iPhone 上做的事

- 添加和编辑提供商
- 填 API Key
- 管理代理
- 导入世界书或大批量记忆
- 深度调整显示系统
- 管理工具审批和 MCP 服务器

## 为什么要这样分

不是因为 Watch 端不行，而是因为：

- 手表输入成本更高
- 深层配置路径本来就更适合大屏幕
- 手机更适合做批量管理和排障

## 一个推荐工作流

1. 在 iPhone 上把提供商、模型和基础偏好配好。
2. 确认同步链路正常。
3. 在 Watch 上把它当成一个已经准备好的“随身交互端”。

## 适合 Watch 的功能组合

### 日常聊天

- 短消息来回
- 已有会话继续聊
- 语音输入补充

### 主动提醒

- 查看 Daily Pulse
- 从通知动作继续进入聊天

### 轻量消费内容

- 读较短回答
- 快速浏览重点
- 在必要时转去手机做更深操作

## 不建议的做法

- 第一次就在手表上配置所有提供商
- 把复杂的工具实验放在 Watch 上做首轮测试
- 在没有同步稳态前双端同时改很多配置
