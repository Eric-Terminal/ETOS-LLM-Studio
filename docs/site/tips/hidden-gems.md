# 隐藏技巧

这一页专门写给「我知道这软件功能很多，但很多东西如果没人提醒，真的不会自己发现」的人。

先说**最重要的一条**——ETOS 里所谓「隐藏技巧」，**很多时候不是藏在二级页面里，而是藏在手势里**。

## 手势相关

### 1. 永远先尝试「长按 + 左右滑」

看到一个**条目、卡片、消息、列表项、记录项**时，不要只会点进去。靠谱的探索顺序是：

1. **先点一次**，看正常主流程
2. **再长按**，看上下文菜单有什么
3. **再左滑 / 右滑**，看有没有快捷操作

这条规则在 ETOS 里**非常通用**，特别是这些位置：

- 会话列表（长按 → 移动 / 重命名 / 删除）
- 提供商和模型列表
- 记忆与会话摘要（左滑删除、右滑归档）
- 世界书条目
- Daily Pulse 反馈历史
- 文件和日志列表
- 聊天消息（长按是最大的彩蛋区）

如果你觉得"这软件应该不至于只有一个点按动作吧"，那大概率就该**先长按一下，再左右划一下**。

### 2. 聊天页消息长按 = 隐藏的功能宝箱

聊天页每条消息长按会出现 8-10 个操作，包括：

- **创建提示词分支**（仅分支系统提示词，或同时复制消息历史）
- **导出到此消息（含上文）**——把对话截断后导出
- **Token 信息** + **思考耗时**——给推理模型用的元数据
- **编辑消息**（包括改 AI 自己的回答）
- **引用**——把这条消息作为下一轮输入的引用

详见 [第一次发起聊天](/guide/first-chat)。

### 3. 会话列表搜索框是全文搜索，不只是标题

会话列表顶部的搜索框（占位文字「**搜索会话标题或消息**」），实际上**对消息正文也做全文搜索**。

点搜索结果会**直接跳到那条历史消息所在位置**——比从头滚省事得多。

## 配置相关

### 4. 先用一个模型打通全链路，再拆角色

ETOS 允许你给聊天、TTS、Daily Pulse、STT、嵌入等**分别选模型**。但第一次使用时，**先用一个稳定模型把主链路走通**，再开始拆角色——能省 90% 的排查时间。

### 5. 工具不是越多越好

很多人第一次看到 MCP / 本地工具 / Shortcuts / Skills **会很兴奋然后一次全开**。结果：

- 模型上下文被工具描述吃掉一半
- 出问题时不知道是哪个工具的锅
- 审批气泡狂弹打扰使用

更实用的做法：

- **日常会话只保留必要工具**
- **工作流会话再专门启用一组工具**
- **想清楚审批策略再决定是否自动执行**

### 6. 记忆 vs 世界书的简单判断法

| 这条信息会… | 放在 |
| --- | --- |
| 被长期反复使用，像知识块 | **记忆** |
| 只在特定场景/关键词出现时才有用 | **世界书** |

后续维护成本会低很多。

### 7. 多 API Key 用英文逗号分隔

提供商的 API Key 字段支持多 Key 轮询：

```
sk-aaaaaaaaaa,sk-bbbbbbbbbb,sk-cccccccccc
```

每次请求自动轮换，某 Key 触发 429 自动跳过。免费 Key / 试用 Key 拼起来够用很久。

### 8. `{api_key}` 是占位符，不是字面值

「请求头覆盖」和"MCP 请求头覆盖"里支持 `{api_key}` / `{token}` 占位符。它们会被替换成当前实际使用的 Key：

```
Authorization=Bearer {api_key}
X-Custom-Auth=xxx-{api_key}-yyy
```

适合接非标准鉴权位的服务。

## 使用习惯

### 9. Daily Pulse 真正的关键是「反馈」

如果你只是**看卡片不给反馈**，它的进化速度会很慢。

**点赞 / 降权 / 隐藏 / 保存为会话** 这四个按钮比频繁换模型更能改变后续质量。**保存为会话**是最强的正反馈信号。

### 10. 「明日想看什么」是给下一次的，不是这次的

Daily Pulse 页面有两个文本输入：

- **当前关注焦点** → 长期方向，每次生成都参考
- **明日想看什么** → **只对下一次生成生效**

晚上想到的"明早起来要看这个"，写进"明日想看什么"。

### 11. 调试工具能让你少走很多 UI 路

当你已经知道某个问题可能和**文件、配置或请求**有关时，优先用 **设置 → 拓展功能 → 应用日志 / 高级诊断 / 反馈助手**。

它们让你**直接看数据**，不必在界面里盲猜。

### 12. 导入导出是迁移手段，不只是备份

ETOS 数据包除了备份还适合：

- **换设备**
- **跨账号 / 跨工作环境迁移**
- **大改动前先保存稳态版本**——遇到问题一键回滚

## Apple Watch 相关

### 13. 让 iPhone 替 Watch 做准备

舒服的双端工作流是：

1. **iPhone**：配模型、工具、显示偏好
2. **同步**：把环境带到 Watch
3. **Watch**：专注聊天、提醒、快速继续

**不要试图在 Watch 上完成复杂配置**——屏幕太小，效率极低。

## 终极建议

### 14. 建立你自己的「使用分层」

很推荐定一个简单分层：

| 用途 | 配置 |
| --- | --- |
| **日常聊天** | 最稳的模型 + 最少的工具 + 自动批准记忆写入 |
| **专项折腾** | 高级参数 + 全工具 + 必要时 MCP / Skills |
| **碎片时间（Watch）** | 接收提醒 + 短轮聊天 + 不做配置 |

这样功能再多，也不会变成一锅乱炖。

### 15. 找不到时，先想"这是治理还是执行"

ETOS 主屏只有"聊天 + 设置"两个 Tab：

- **执行**类操作（聊天、附件、模型切换） → 聊天页
- **治理**类操作（提供商、工具、记忆、世界书、同步、Daily Pulse 配置） → 设置 Tab

按这条规则定位入口，**命中率 90%+**。
