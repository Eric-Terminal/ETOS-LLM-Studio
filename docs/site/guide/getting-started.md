# 快速开始

如果你是第一次接触 ETOS LLM Studio，建议不要一上来就把所有高级功能都开满。  
最顺手的方式是先把“能稳定聊天”这条链路跑通，再逐步开启工具、记忆、世界书和同步。

## 三步进入可用状态

| 步骤 | 目标 | 建议 |
| --- | --- | --- |
| 1 | 跑起 App | 先在 iPhone 或模拟器上打开工程，确认工作区能启动 |
| 2 | 配一个提供商 | 先接一个你最熟悉的 OpenAI / Claude / Gemini 或兼容接口 |
| 3 | 发起第一轮对话 | 先做纯文本聊天，确认返回正常，再继续折腾语音、图片和工具 |

## 最短上手路径

1. 按 [安装与运行](/guide/installation) 把工程拉起来。
2. 按 [第一次配置提供商](/guide/first-provider) 先加一个稳定模型。
3. 按 [第一次发起聊天](/guide/first-chat) 建立第一个会话。
4. 配通之后，再看 [界面导览](/guide/interface-tour) 和 [隐藏技巧](/tips/hidden-gems)。

## 你会先接触到哪些能力

### 聊天与模型

- 多提供商与多模型切换
- 自定义请求头、参数表达式、原始 JSON 请求体
- 图片、语音、导入导出和富文本渲染

### 工具与自动化

- MCP 服务器接入
- 本地工具、文件操作与审批策略
- Skills、Shortcuts 与 URL Scheme 路由

### 长期使用价值

- 本地 RAG 记忆与主动检索
- 世界书与条件注入
- Daily Pulse 主动情报卡片
- iPhone 与 Apple Watch 双端同步

## 建议的使用顺序

### 先做什么

- 把一个模型调通
- 建一个普通会话
- 熟悉聊天页和设置页的基本入口

### 暂时别急着做什么

- 一上来导入大量记忆
- 同时启用太多工具和 MCP 服务器
- 在手表上完成大量提供商配置

这些功能不是不能用，而是它们更适合在你已经熟悉主链路之后再开。
