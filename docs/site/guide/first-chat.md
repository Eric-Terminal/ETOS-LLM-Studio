# 第一次发起聊天

## 先把目标定简单一点

第一轮聊天只需要验证三件事：

- 会话能正常创建
- 消息能正常发送
- 模型能正常返回文本

## 建议的第一条消息

尽量发一条纯文本、很短、没有附件也没有工具依赖的话。  
例如简单自我介绍、翻译一句话、总结一段短文本，这种最容易排查。

## 如果第一轮就成功了

你可以继续尝试这些扩展能力：

- 给会话分文件夹
- 搜索历史聊天
- 测试消息定位与序号跳转
- 发送图片或语音
- 导出 Markdown / TXT / PDF

## 如果第一轮没成功

优先按照这个顺序排查：

1. 提供商配置是否正确
2. 当前会话是否选中了正确模型
3. 网络、代理、鉴权是否通畅
4. 是否额外启用了会影响请求的复杂参数

## 发起聊天后可以顺手观察什么

### 渲染体验

- Markdown 是否正常显示
- 代码块高亮和复制反馈是否正常
- 长文本、引用块、Mermaid 或数学公式是否符合预期

### 会话管理

- 会话是否能被搜索到
- 是否能移动到文件夹
- 是否能跨端继续发送

### 音频与多模态

- 语音输入是否能回填
- 图片是否能发送给当前模型
- 朗读是否能切到你想要的 TTS 模型

## 什么时候该开始用工具

当纯文本聊天稳定以后，再开启 MCP、Shortcuts、本地工具或文件系统工具。  
这样一旦行为异常，你能更容易区分是模型链路问题，还是工具链路问题。
