# 界面导览

ETOS LLM Studio 主界面只有两个 Tab：**聊天**和**设置**。看似很少，但「设置」是整台 AI 工作台的总控台，里面有 20+ 个子入口。这一页把整张地图画给你，让你不再为"某个功能明明有但找不到"发愁。

## 聊天 Tab

你日常用得最多的界面。它的物理布局是这样的：

```
顶部导航栏：[菜单] 「新的对话」标题 [模型选择] [更多]
↓
中央：会话区（消息气泡）
↓
底部输入区：[+] [输入框]  [工具开关]  [发送]
```

**这里能干什么**（按使用频率排序）：

- 发文字 / 图片 / 语音 / 文件
- 切换当前会话的模型（右上角）
- 长按某条消息 → 复制、编辑、删除、导出、创建分支
- 左上角菜单 → 会话列表（搜索 / 文件夹 / 新建）
- 输入框附近的小图标开关：联网搜索、MCP 工具、Skills、快捷指令——**只对下一次发送生效**

::: tip 长按消息几乎能做所有事
ETOS 把每条消息相关的操作都收在长按菜单里：复制、引用、编辑、删除、Token 信息、思考过程详情、导出、分支……找不到某项操作时先长按试试。
:::

## 设置 Tab

进入「设置」后是一张分组列表，**严格按下面这个顺序**排列：

### Section 1 · 当前模型

| 入口 | 干什么 |
| --- | --- |
| **模型** | 选当前所有新会话默认用的那个 AI。所有「已启用」的模型都会出现在这里。 |
| **开启新对话** | 直接新建一条会话并跳回聊天 Tab。 |

::: warning "暂无可用模型"
如果这里显示"暂无可用模型，请先在'提供商与模型管理'中启用。"，说明你的提供商虽然加了，但模型行没开启用开关。
:::

### Section 2 · 对话行为

| 入口 | 干什么 | 详细教程 |
| --- | --- | --- |
| **历史会话管理** | 跨会话搜索、按文件夹整理、批量移动 / 删除 | [聊天与模型](/modules/chat-and-models) |
| **提供商与模型管理** | 增删改提供商、配 API Key、拉模型列表、连通性测试 | [第一次配置提供商](/guide/first-provider) |
| **偏好设置** | 全局系统提示词、温度 / top_p、流式开关、最大历史消息数、是否注入系统时间 | [聊天与模型](/modules/chat-and-models) 进阶选项 |
| **语音朗读（TTS）** | 设默认 TTS 模型、朗读语速 | [聊天与模型](/modules/chat-and-models) |

### Section 3 · 拓展能力

这是设置页里**最长的一组**，所有"额外能力"都收在这里：

| 入口 | 干什么 | 详细教程 |
| --- | --- | --- |
| **工具中心** | 模型可调用的本地工具汇总（计算器、文件操作、HTTP 请求等），含审批策略 | [工具与 MCP](/modules/tools-and-mcp) |
| **每日脉冲** | 每天定时主动生成"今日值得看什么"卡片 | [每日脉冲](/modules/daily-pulse) |
| **用量统计** | 按提供商 / 模型 / 日期看你的 Token 用量和花费 | [聊天与模型](/modules/chat-and-models) |
| **记忆系统** | 跨会话长期记忆，让 AI 记住你的偏好 / 项目背景 | [记忆与世界书](/modules/memory-worldbook) |
| **MCP 工具集成** | 接 Model Context Protocol 服务器（GitHub、文件系统、搜索等） | [工具与 MCP](/modules/tools-and-mcp) |
| **Agent Skills** | 导入本地 Skill 包，让模型获得专项能力（如写 SQL、画图） | [Skills 与快捷指令](/modules/skills-and-shortcuts) |
| **快捷指令工具集成** | 把 iOS Shortcuts 暴露给模型当工具用 | [Skills 与快捷指令](/modules/skills-and-shortcuts) |
| **图片生成** | 配 DALL·E / Gemini Imagen 等图像生成模型 | [聊天与模型](/modules/chat-and-models) |
| **世界书** | 按关键词触发的"补丁式"知识注入 | [记忆与世界书](/modules/memory-worldbook) |
| **语音输入** | 配 STT（语音转文字）模型，支持把语音直接当多模态附件发送 | [聊天与模型](/modules/chat-and-models) |
| **拓展功能** | 各种实验性 / 调试用功能开关 | [调试与反馈](/modules/debug-feedback) |

::: info 这些功能彼此独立
拓展能力里的每一项都可以**独立开关**。比如你只想要每日脉冲，可以完全不碰 MCP 和世界书；反之亦然。**不要看到列表长就害怕，按需开启即可**。
:::

## Apple Watch 上看什么

Watch App 是 iPhone 的精简版。它**不提供完整配置 UI**，主要功能：

- **新建对话** → 用配过的提供商和模型直接发消息
- **历史会话列表** → 看同步过来的对话
- **接收 Daily Pulse 推送** → 在通知里读今天的卡片
- **语音输入** → 长按表冠或抬腕说话直接发送

Watch 上**没有**：提供商配置、世界书编辑、MCP 管理、Skills 管理。这些都在 iPhone 配好后通过同步带过去。

## 一个找不到时的通用策略

碰到"我记得有这个功能但不知道在哪"时按这个顺序找：

1. **先看是不是消息相关** → 长按聊天里的某条消息
2. **看是不是会话相关** → 左上角菜单 → 会话列表
3. **看是不是配置相关** → 设置 Tab → 拓展能力（最大概率藏在这里）
4. **看是不是显示相关** → 设置 → 显示与体验
5. **看是不是导入导出 / 跨端** → 设置 → 显示与体验 → 同步与备份
6. **实在找不到** → 直接在文档站搜索框搜关键词
