# 界面导览

ETOS LLM Studio 的很多能力都不是首页大按钮式展示，而是藏在会话、设置、工具中心和各类子页面里。  
这一页的目标就是帮你建立一张“功能地图”。

## 聊天页

聊天页是你最常停留的地方，但它不只是消息列表。

### 你通常会在这里完成

- 新建或切换会话
- 发送文本、图片、语音
- 查看 Markdown、代码块、引用、Mermaid
- 调整当前会话关联的模型或工具
- 导出当前会话

### 它背后挂着什么能力

- 会话搜索和定位
- 文件夹分类与批量移动
- 单会话跨端发送
- TTS 朗读与 STT 回填

## 设置页

设置页其实是整台“AI 工作台”的总控台。

### 这里通常包含

- 提供商与模型配置
- 显示系统与字体回退
- 记忆与世界书
- 工具中心与 MCP
- Daily Pulse
- 同步、备份、代理和反馈

如果你觉得“某个功能明明有但不知道在哪开”，十有八九就在设置页的某个子入口里。

## 工具中心

工具中心负责统一治理聊天中可调用的能力。

### 这里值得特别注意的点

- 工具不只一种，MCP、Shortcuts、本地工具会放在一起管理
- 你可以控制工具是否默认启用
- 审批策略会直接影响聊天时工具能否自动执行

## 记忆与知识相关入口

这部分通常会被低估，但它们是“长期使用体验”最关键的一块。

### 你可以在这里找到

- 本地 RAG 记忆
- 主动检索开关
- 会话记忆与用户画像
- 世界书导入、绑定与触发策略

## Daily Pulse

如果你想让应用不只是等你提问，而是主动整理“今天值得看的东西”，这一块就是入口。

你可以在这里设置：

- 生成模型
- 提醒时间
- 卡片反馈
- 继续聊或转任务

## 同步、调试与反馈

这部分更偏维护和高级使用，但很实用。

### 同步与备份

- iPhone ↔ Apple Watch 数据互通
- 导出 ETOS 数据包
- 启动保护备份与恢复

### 调试与反馈

- 局域网调试
- 请求日志与测速分析
- 应用内反馈助手和工单状态

## 一个实用建议

如果你在聊天页里找不到入口，不要只盯着当前会话。  
ETOS LLM Studio 把很多“会影响整机行为”的能力放在设置页和工具中心，是为了让聊天页保持可用，而不是把所有控制都挤进主界面。
