# 同步与备份

ETOS LLM Studio 不是把 iPhone 和 Apple Watch 做成两个彼此独立的 App。  
它内置了完整的双端同步链路，并且把导入导出、启动备份和恢复也纳入了整体设计。

## 跨端同步

### 已覆盖的主要数据

- 提供商配置
- 会话
- 世界书
- 工具配置
- Daily Pulse 相关数据

### 同步形态

项目内部提供 Manifest / Delta 差异同步主链路，不只是粗暴地全量覆盖。

## 为什么这很重要

对这类功能很多、配置很深的客户端来说，如果没有可靠同步，你会很快落入：

- 手机上配了一半，手表上又得重来
- 长期资料只能单端可用
- 备份恢复时很难控制粒度

## 导入导出

项目支持 ETOS 数据包导出与导入。  
这意味着你不仅能做双端同步，也能做更明确的手动迁移和备份。

### 典型场景

- 换设备
- 导入到手表
- 手动保存关键配置
- 导出后再用自定义地址 POST 上传

## 启动保护备份

如果你担心数据库损坏或大改配置后回不来，这个能力很重要。

### 价值

- 启动时建立备份点
- 出现损坏时可恢复
- 对长期积累的数据更友好

## 使用建议

### 当你主要在手机上配置

优先把复杂配置都在 iPhone 做完，再同步到 Watch。  
这会比双端同时编辑稳定得多。

### 当你准备做大规模导入

先保留一个可回退的备份点，再批量导入会话、记忆或世界书。
