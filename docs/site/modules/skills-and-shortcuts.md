# Skills 与快捷指令

这一部分决定了 ETOS LLM Studio 能否从“聊天客户端”继续往“可组合的个人 AI 工具台”扩展。

## Agent Skills

Skills 的意义在于把一组专门能力封装成可导入、可启用、可治理的能力单元。

### 当前接入特点

- 技能能力已经贯穿到工具中心
- 可以统一开关管理
- iPhone 端支持从本地文件导入
- watchOS 端支持通过 URL 下载导入

## 这类能力适合做什么

- 给不同场景准备不同工具包
- 为特定会话启用更明确的执行能力
- 把复杂流程整理成可复用入口，而不是每次都手动拼提示词

## Shortcuts 与 URL Scheme

项目也集成了 Apple Shortcuts 体系。  
这意味着 ETOS LLM Studio 不只是自己内部可用，还可以被系统自动化链路调用。

### 可以带来的好处

- 通过快捷指令直接调用 AI 能力
- 自定义工具入口
- 结合系统自动化或外部触发
- 借助 URL Scheme 把结果路由回特定场景

## 适合怎么搭配

### 日常使用

- 用聊天页做主要交互
- 用 Shortcuts 做轻量自动化入口
- 用 Skills 管理更复杂、成套的能力

### 更重的工作流

- 用 MCP 提供外部能力
- 用 Skills 组织调用方式
- 用审批策略控制工具执行边界

## 为什么这块值得单独看

很多项目会说自己“支持工具”，但真正麻烦的是导入、启用、治理和平台差异。  
ETOS LLM Studio 在这部分已经明显超出普通“加一个插件入口”的级别。
