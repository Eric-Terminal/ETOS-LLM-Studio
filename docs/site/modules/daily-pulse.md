# 每日脉冲

普通 LLM 客户端永远在等你提问。**每日脉冲（Daily Pulse）**反过来：每天早上自动生成一组"今天对你来说值得看的事"卡片，主动推给你。

它的信号来源**不是猜的**——是你最近的聊天、记忆、反馈历史、明日策展和（可选）外部 MCP / 快捷指令结果。这一页教你配置它。代码层信号源详见 [每日脉冲设计原理](/design/daily-pulse)。

## 它在什么时候有用

| 场景 | 怎么帮你 |
| --- | --- |
| 早上不知道今天该想什么 | 卡片直接列出"今天可能想跟进的 N 件事" |
| 你在跟进多个项目 | 把分散在不同会话的"未完成事项"汇总成今日待办 |
| 你订了一堆信息源 | 通过 MCP / 快捷指令拉外部信号（新闻、邮件、日历），统一展示 |
| 你想让 AI"学会你" | 通过点赞 / 降权 / 隐藏反馈逐步沉淀偏好 |

如果你只是**偶尔聊几句问问题**，每日脉冲对你不重要，可以跳过。

## 新手必读

### 在哪管

```
设置 → 拓展能力 → 每日脉冲
```

### 第一次配：四步搞定

#### 1. 选模型

每日脉冲的卡片由一个**专门的模型**生成。你需要：

```
设置 → 提供商与模型管理 → 专用模型 → 每日脉冲模型
```

选一个**性价比合理的**模型——它每天只调用一次，但生成的内容比较长（3 张卡片左右）。推荐：

| 模型 | 理由 |
| --- | --- |
| GPT-4o-mini | 便宜、速度快、整理能力够 |
| Claude 3.5 Haiku | 同上 |
| Gemini 2.5 Flash | 同上 |
| 你日常用的旗舰 | 如果你想要更精致的总结 |

#### 2. 开启自动生成

每日脉冲页顶部的开关：

| 开关 | 作用 |
| --- | --- |
| **每日首次打开自动补生成** | 每天第一次打开 App 时，如果今天还没生成会自动补一份 |
| **晨间提醒** | 每天定时弹通知告诉你新一期已经准备好 |
| **提醒时间** | 设定通知时间（默认 8:00） |

#### 3. 第一次手动生成

如果想立刻看效果，点「**立即生成**」（带 ✨ 图标）。

「**当前会优先使用最近聊天、记忆系统、请求日志、反馈历史、明日策展和你的关注焦点，并可选结合外部上下文生成约 3 张卡片。**」

第一次生成时间通常 10-30 秒，模型越慢越久。

#### 4. 看卡片 + 给反馈

生成完会出现「**今天的卡片**」Section。每张卡片有四个反馈按钮：

| 按钮 | 含义 |
| --- | --- |
| ❤️ **喜欢** | 这卡片有用，以后多生成这种 |
| 👎 **降权** | 不太想看，以后减少这种 |
| 🙈 **隐藏** | 这次的我不看，下次不再展示 |
| 🔖 **保存为会话** | 把这卡片转成正式聊天会话，继续往下聊 |

反馈会作为**长期偏好信号**沉淀，影响后续生成。**认真用反馈比频繁换模型更能提升体验**。

### 关注焦点 + 明日策展

页面中部有两个文本输入区，是你**直接告诉 Daily Pulse 想看什么**的入口：

#### 当前关注焦点

> 例如：继续推进某个项目、帮我整理下一步、关注最近反复提到的话题

写在这里的内容会作为**最高优先级**信号参与下一次生成。适合长期持续的关注方向（比如"现在主要做 XX 产品的发布筹备"）。

#### 明日想看什么

> 例如：明天优先帮我跟进 PR、安排会议、看某个项目的下一步

写在这里的内容**只对下一次生成生效**——下一次每日脉冲到达目标日期生成时，会优先纳入这段策展。

适合：晚上想到的"明早起来要看这个"。

### Pulse 任务

如果某张卡片你觉得需要持续跟进，可以把它转成**Pulse 任务**。这些任务：

- **跨天保留**
- 在每次每日脉冲生成时作为「**还需要推进的事情**」参与策展
- 完成后可以「**标记完成**」或「**移除**」
- 完成的任务可以批量「**清理已完成任务**」

::: tip Pulse 任务 vs 关注焦点
- **关注焦点**：长期方向，比如"做 XX 产品"
- **Pulse 任务**：具体待办，比如"修复 issue #123"
:::

### 反馈历史

「**反馈历史会作为长期偏好信号保留**」——你历史上对所有卡片的点赞 / 降权 / 隐藏 / 保存都会沉淀，参与未来生成的策展。

可以在「**查看完整反馈历史**」里：

- 看每条历史反馈的卡片标题、反馈类型、发生日期
- 删除单条（如果某次点错）
- 整体清空（如果想"重启"AI 对你的理解）

## 进阶选项

### 外部上下文

ETOS 可以把**外部信号**也纳入每日脉冲生成。设置在「**外部上下文**」Section：

| 开关 | 作用 |
| --- | --- |
| **纳入 MCP 服务器能力** | 让生成时知道你接了哪些 MCP，可以参考它们近期返回的数据 |
| **纳入快捷指令能力** | 同上，参考最近快捷指令工具的结果 |
| **纳入最近外部结果** | 把最近 MCP / 快捷指令的实际输出作为材料 |
| **纳入公告与趋势信号** | 把系统公告、最近热门话题等趋势性信息纳入 |

**典型用法**：

- 你接了「读最新邮件」的 MCP → 早上的 Daily Pulse 自动包含"今天最值得回的几封邮件"
- 你写了「今日 Hacker News」的 Shortcut → 早上脉冲自动包含技术圈热点
- 你接了「查日历」的 MCP → 卡片包含"今天的会议安排"

::: warning 外部信号要靠你主动构建
ETOS 不会**凭空知道**外部事实。Daily Pulse 只能用：你**已经聊到过的事**、**已经写入记忆的事实**、以及**最近通过 MCP / 快捷指令实际拉回来的数据**。

如果你完全不接 MCP 也不写 Shortcut，外部信号开关意义不大。
:::

### 后台主动送达

「**主动送达**」开关组允许 ETOS 在你**没打开 App** 时也自动调度生成（依赖 iOS 后台刷新权限）。

- iOS 系统会根据使用模式决定**实际何时执行**——你不能精确指定时刻
- 设了"晨间提醒 7:00"不一定 7:00 准点跑，但通常在合理范围内（早 8 点前）

### 准备状态

正在生成时页面会显示「**正在为你准备今天的每日脉冲**」，并告诉你「**系统已在 HH:MM 开始准备今天这一期**」。

如果生成失败，会显示错误信息并允许重试。

### 卡片全被隐藏怎么办

如果你把今天的卡片全部隐藏了，会显示「**这次生成的卡片都被你隐藏了。你可以重新生成一份新的每日脉冲。**」——直接点重新生成。

不过频繁全部隐藏会被 AI 学到"这种风格不喜欢"，建议**用降权而非隐藏**来微调方向。

### Watch 端 Daily Pulse

Apple Watch 上：

- **接收通知**：晨间提醒会推到 Watch
- **快速查看**：通知里直接展开卡片摘要
- **不能编辑反馈**：详细的点赞 / 降权要回 iPhone

适合：醒来抬腕扫一眼"今天的概览"，详细处理回 iPhone。
