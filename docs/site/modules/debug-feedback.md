# 调试与反馈

ETOS LLM Studio 对“遇到问题以后怎么排查”这件事考虑得比普通客户端深很多。  
它不是只有一个简陋的报错弹窗，而是把局域网调试、反馈助手、请求日志和网络代理都做成了完整链路。

## 局域网调试

项目内置局域网调试客户端，并且仓库里提供了配套的电脑端调试工具。

### 这类能力能做什么

- 浏览应用内文件
- 管理会话数据
- 上传下载调试资料
- 捕获请求
- 在浏览器里使用 Web 控制台

这对“功能很多但入口较深”的项目尤其重要，因为你不必只靠 UI 一层层点进去排查。

## 请求日志与测速分析

项目会维护独立请求日志，并提供细分 Token 汇总与流式速度统计。  
如果你在做这些事情，这块很有帮助：

- 对比不同模型的响应速度
- 观察流式输出体验
- 定位某个提供商是否异常

## 应用内反馈助手

反馈系统不是简单跳浏览器发 issue。

### 它已经具备

- 反馈分类
- 环境信息采集
- PoW 提交链路
- 工单评论对话
- 开发者标记展示
- 状态自动刷新
- 双端同步

## 网络代理

如果你的使用环境需要代理，项目内也有正式支持。

### 当前方向

- 全局 HTTP(S) 代理
- 提供商级 HTTP(S) 代理
- SOCKS 代理
- 鉴权支持

## 什么情况下先看这一页

- 你已经能用，但怀疑某个链路不稳定
- 你需要定位模型、代理、工具或网络问题
- 你想更高效地给开发者反馈问题
