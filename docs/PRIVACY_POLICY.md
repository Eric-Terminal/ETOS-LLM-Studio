# ETOS LLM Studio 隐私政策

**生效日期：** 2026年3月22日

## 1. 前言

ETOS LLM Studio 采用 **“本地优先（Local-first）”** 的设计思路。大多数核心数据会保存在您的设备本地，由您自己掌控。

我们**不会在开发者控制的服务器上**默认存储您的聊天内容、语音录音、图片附件或 API Key；但应用本身确实会在设备本地保存聊天记录、配置、记忆、请求日志等数据，以支撑同步、统计、Daily Pulse、年度汇总等功能。

## 2. 设备本地存储与本地处理

### 2.1 本地保存的数据
- **聊天与会话数据：** 聊天记录、会话配置、临时草稿、图片/音频/文件附件引用等会保存在应用沙盒内。
- **知识与个性化数据：** RAG 记忆、世界书、Daily Pulse 数据、反馈历史、任务等会保存在设备本地。
- **本地请求日志：** 应用会在本地记录模型请求的元数据，例如提供商名称、模型标识、请求时间、完成时间、状态、是否为流式请求、Token 用量等。**这些日志不包含聊天正文**，主要用于 API 流量分析、响应测速、Daily Pulse 统计、年度汇总等设备端功能。
- **反馈工单缓存：** 如果您使用反馈助手，应用会在本地保存工单编号、票据令牌、标题、状态、更新时间和公开链接等信息，方便后续查看处理进度。

### 2.2 凭据存储
- **API 提供商配置：** 提供商配置本身保存在本地。
- **API Key：** API Key 使用系统 **Keychain** 保存，而不是写入普通 JSON 配置文件。
- 如果您启用了 Apple 的相关同步能力，API Key 可能通过 **iCloud 钥匙串** 在您自己的设备之间同步。

## 3. AI 请求、音频与图片

### 3.1 文本聊天
- 您发送给模型的文本消息会**直接**发送到您自行配置的第三方 AI 提供商（如 OpenAI、Anthropic、Google、兼容 OpenAI 的服务或自建模型）。
- 我们不会在开发者服务器上拦截、转存或训练这些聊天内容。

### 3.2 语音与音频
- 应用仅在您主动点击录音或语音输入时访问麦克风。
- 录音会作为音频附件发送，或交给您配置的语音/转写模型处理。
- 音频不会默认发送给开发者服务器。

### 3.3 图片与文件
- 应用仅在您主动选择图片或文件时访问相关内容。
- 您附加的图片/文件会发送到当前模型提供商或您主动连接的调试端，不会默认上传到开发者服务器。

## 4. 同步、本地网络与 Apple 服务

### 4.1 iPhone / Apple Watch 同步
- iPhone 与 Apple Watch 之间的实时同步通过 **WatchConnectivity** 完成。
- 这类同步属于 Apple 提供的设备间通信能力，不经过开发者自建服务器。

### 4.2 iCloud 同步
- 如果您在应用中主动开启 **iCloud 同步**，所选数据会通过 Apple 的 iCloud / CloudKit 体系在您自己的设备之间同步。
- API Key 不会写入普通同步包，而是依赖 **iCloud 钥匙串**（如果系统与账户允许）。

### 4.3 本地网络与调试
- 应用可能请求本地网络权限，用于可选的 **局域网调试 / 远程文件访问** 功能。
- 如果您主动连接自己的电脑调试工具，相关应用文件或请求流量可能会发送到**您自己控制的本地调试端**，用于查看日志、分析请求或管理应用文件。

## 5. 反馈助手

### 5.1 何时会向开发者侧服务发送数据
仅当您**主动使用反馈助手并提交反馈**时，应用才会向反馈服务发送数据。

### 5.2 提交反馈时可能发送的数据
- **您主动填写的内容：** 反馈类型、标题、详细描述、复现步骤、预期行为、实际行为、补充说明。
- **环境快照：** 平台（iOS / watchOS）、App 版本号、Build 号、操作系统版本、设备型号标识、语言地区、时区。
- **极简诊断信息：** 时间戳、当前配置的提供商数量、非临时会话数量、平台信息、App 版本信息。
- **防滥用校验数据：** 为了防止垃圾提交，反馈请求还会附带 challenge、时间戳、签名和 PoW 相关字段。

### 5.3 不会自动附带的内容
- 不会自动附带您的完整聊天记录。
- 不会自动附带 API Key。
- 不会自动把本地请求日志整包上传。

### 5.4 脱敏与提醒
- 反馈文本在提交前会做一层**基础脱敏**，尝试遮盖常见的 Bearer Token、API Key 字样和部分常见密钥格式。
- 但这不是完整的数据防泄漏系统。**请不要在反馈内容中主动填写密钥、身份证号、住址、健康信息等敏感内容。**

### 5.5 工单状态刷新
- 如果您之后在反馈中心查看工单状态，应用会使用本地保存的工单编号与票据令牌向反馈服务查询处理进度和评论。

## 6. 第三方服务与开发者服务器边界

当前版本中，可能涉及的数据处理方主要包括：
1. **您自行配置的 AI 提供商**（负责处理聊天、图像、语音等模型请求）
2. **Apple 提供的系统服务**（如 WatchConnectivity、iCloud / CloudKit、iCloud 钥匙串）
3. **开发者运营的反馈服务**（仅在您主动提交或查询反馈工单时使用）

除反馈助手这种**用户主动触发**的场景外，我们不会把您的聊天内容、本地记忆、本地请求日志、年度汇总数据或 API Key 自动上传到开发者控制的服务器。

## 7. 分析、广告与崩溃信息

- 当前代码中**未集成第三方广告 SDK**。
- 当前代码中**未发现通用型开发者侧埋点/行为分析 SDK**。
- 如果您在 iOS 系统层面选择与开发者共享分析，Apple 可能会按照其自身规则收集匿名崩溃日志或诊断信息。

## 8. 免责声明

本应用是一个通用 LLM 客户端工具。

1. **中立工具属性：** 应用可能预置部分服务端点或反馈服务地址，但默认不内置可直接使用的商业模型凭据。
2. **内容责任：** 所有 AI 生成内容均由您自行配置的第三方服务提供商返回，我们无法保证其准确性、合法性或适用性。
3. **服务选择：** 您选择使用的任何 API 服务或反馈服务，都应由您自行评估其隐私政策、条款、成本与风险。
