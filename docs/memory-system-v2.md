# 记忆系统 v2 架构

## 目标

新版记忆系统不再把所有内容视为同质文本。它需要同时回答三个问题：这是什么类型的信息、它现在是否仍然有效、它为什么应当被召回。

系统保留现有的本地 SQLite、JSON 镜像和向量索引，不引入新的服务端依赖。旧记忆与旧画像会在读取时获得默认元数据，无需用户手动迁移。

## 参考思路

- [Mem0](https://github.com/mem0ai/mem0)：多信号检索、实体关联、时间推理，以及将助手已经确认完成的行为作为一等记忆。
- [Graphiti](https://github.com/getzep/graphiti)：事实有效期、来源追踪和保留被新事实取代的历史记录。
- [Letta](https://docs.letta.com/guides/core-concepts/memory/memory-blocks)：始终可见的结构化用户画像与按用途划分的记忆块。
- [LangMem](https://langchain-ai.github.io/langmem/concepts/conceptual_guide/)：区分语义、情景和程序记忆，并让重要度、近期使用与使用强度参与召回。

项目没有直接移植完整知识图谱。当前个人本地记忆规模下，图数据库会显著增加迁移、同步和 watchOS 资源成本；实体字段与时间有效期已经覆盖现阶段最有价值的图记忆能力。

## 数据分层

### 长期记忆条目

每条 `MemoryItem` 包含：

- 类型：事实、偏好、经历、长期规则；
- 来源：手动、用户陈述、助手行为、会话摘要或导入；
- 重要度与置信度；
- 相关实体；
- 生效与失效时间；
- 来源会话；
- 访问次数与最近访问时间。

归档表示用户主动遗忘。有效期则表示事实仍被保留，但只在历史或未来时间查询中参与召回。

### 情景记忆

会话摘要继续按会话保存。它用于跨对话连续性，不和稳定事实混成同一个向量集合。

### 用户画像

画像由一个可手动编辑的概览和若干结构化事实组成。事实按身份、沟通、专长、兴趣、工作方式、目标、约束、关系等分类，并保留置信度、证据次数、观察时间与来源会话。

模型更新画像时必须输出固定 JSON schema。若模型或用户自定义提示词仍返回普通文本，系统会兼容保存为旧式概览，不会丢失已有结构化事实。

## 检索流程

默认聊天检索使用 `hybrid` 模式：

1. 从向量索引取得较宽的语义候选集；
2. 对本地原文计算饱和词频分数；
3. 匹配查询中出现的实体；
4. 根据“现在、以前、未来”等表达选择有效时间窗口；
5. 融合重要度、置信度、近期访问与累计使用强度；
6. 按父记忆去重后返回 Top K。

当前权重为：语义 42%、词法 25%、实体 12%、重要度 8%、置信度 5%，剩余权重用于近期性、使用强度、时间和类型。嵌入服务不可用时，语义通道权重自然归零，本地检索仍可继续工作。

访问次数只在 1、2、4、8 等里程碑或跨日时立即持久化，减少聊天期间的数据库与跨设备同步写入。

## 写入与更新

`save_memory` 接受类型、来源、实体、重要度、置信度和有效期。旧调用只提交正文时仍可正常保存。

当事实变化时，不覆盖旧内容：先通过 `edit_memory.valid_until` 结束旧事实的有效期，再写入带有 `valid_from` 的新事实。这样“现在”和“以前”的查询能得到不同答案。

仅修改元数据时不会重新生成嵌入；正文发生变化时才更新向量索引。

## 持久化与同步

记忆数据库迁移新增结构化元数据列，画像表新增事实 JSON 和 schema 版本。JSON 镜像及同步包沿用 `Codable`，因此新增字段会随现有备份和设备同步链路传输。

多端画像合并会按“分类 + 规范化事实文本”去重，合并来源会话、时间、置信度和证据次数；概览冲突仍会标记为待模型整理。
